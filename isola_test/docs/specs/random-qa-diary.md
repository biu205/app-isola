# 隨機問答日記功能規格

## 基本資訊
- **功能名稱**：隨機問答日記（Random Q&A Diary）
- **版本**：v1.1
- **狀態**：實作中
- **優先度**：高
- **建立日期**：2026-05-31

## 目的
透過每日隨機呈現的問卷，引導使用者從多個維度記錄情緒與生活片段，降低「不知道寫什麼」的焦慮，同時保持記錄的多元性與趣味性。

## 範圍

### In-Scope（包含）
- 18 種日記類型系統
- 每日隨機題目選擇（2 選 1）
- 題目庫管理（本地 + 雲端同步）
- 題目回答介面與存儲
- 月度題目統計

### Out-of-Scope（不包含）
- 自訂題目功能（階段二）
- 題目難度調整（階段二）
- 社群分享（階段二）

## 18 種日記類型

| 編號 | 類型 | 描述 | 示例 |
|------|------|------|------|
| 1 | 正常日記 | 今天發生了什麼？ | 「今天和朋友聚餐，很開心」 |

| 3 | 音樂 | 今天聽了什麼音樂？ | 「重複聽了《月亮代表我的心》」 |
| 4 | 影片 | 今天看了什麼電影/影集？ | 「看完第三季《黑鏡》」 |
| 5 | 圖像 | 上傳一張今天的相片 | [圖片] |
| 6 | 穿搭 | 今天穿了什麼？感受如何？ | 「穿新買的外套，自信心 +100」 |
| 7 | 食物 | 今天吃了什麼美食？ | 「嘗試新的湯咖哩，好吃！」 |
| 8 | 鐵鋁罐 | 今天喝了什麼飲料？ | 「無糖美式 2 杯」 |
| 9 | 成就 | 今天完成了什麼？ | 「終於交出企劃案」 |
| 10 | 小確幸 | 今天最開心的小事 | 「咖啡店老闆主動多給一份甜點」 |

## 使用者故事

### 故事 1：每日隨機挑戰
**As a** 想保持日記新鮮感的使用者  
**I want to** 每天看到不同的問題，從不同角度回答  
**So that** 我不會感到記錄的乏味

### 故事 2：自由選擇
**As a** 有特定記錄習慣的使用者  
**I want to** 能從 2 個隨機題目中選擇我想回答的那個  
**So that** 我有自主權，不被強制回答不想談的話題

### 故事 3：多媒體記錄
**As a** 喜歡用照片記錄生活的使用者  
**I want to** 能上傳圖片作為回答  
**So that** 我的日記更生動有趣

## 功能流程

```
1. 使用者進入首頁
2. 系統根據當天日期隨機選擇 2 個題目
3. 介面顯示：「今天的問題是...」
4. 顯示題目 A 與題目 B，使用者選擇其一
5. 進入回答介面（文字輸入或圖片上傳）
6. 使用者完成回答
7. 系統儲存回答內容
8. 顯示「已記錄」狀態或「選擇另一題」選項
9. 日記存儲至 SwiftData 與 Firestore
```

## 驗收標準

- [ ] 首次打開 App 當日顯示 2 個隨機題目
- [ ] 使用者能清楚看到題目與選項
- [ ] 點擊題目後進入對應的回答介面
- [ ] 文字類題目有輸入框可編輯
- [ ] 圖片類題目可上傳或拍照
- [ ] 回答內容可本地保存（草稿）
- [ ] 提交後日記內容正確存儲
- [ ] 同一天內不會重複顯示相同題目
- [ ] 隔天自動更新為新的 2 個題目
- [ ] 題目庫更新時，App 自動同步（無感更新）

## 技術實現

### 題目庫結構
```swift
struct QuestionTemplate {
    let id: UUID
    let type: DiaryType // 18 種類型中的其中一個
    let category: String
    let prompt: String
    let inputType: InputType // text, image, multiline, etc.
    let version: Int // 題庫版本號
}

enum DiaryType: String, CaseIterable {
    case normal, emotion, music, video, image, outfit, food, drink, achievement, 
         smallJoy, fun, gratitude, observer, crossMoment, fiveSenses, learning, 
         hypothetical, qa
}
```

### 隨機選擇演算法
```
1. 根據當天日期生成種子（確保同一天選擇相同題目）
2. 從題庫中隨機選擇 2 個不同類型的題目
3. 如果使用者已回答過該類型，降低重複概率
4. 儲存選擇結果至本地，隔天更新
```

### 題庫同步
- App 啟動時檢查 Firestore 中的題庫版本
- 若版本更新，自動下載新題庫
- 本地快取以支援離線使用
- 同步時間戳記防止衝突

## 測試要點

### 單元測試
- [ ] 隨機選擇演算法返回 2 個不同題目
- [ ] 同一天內選擇保持一致
- [ ] 題庫版本比對邏輯正確

### UI 測試
- [ ] 題目文字清晰可讀
- [ ] 選擇按鈕可點擊與反應迅速
- [ ] 回答介面根據題目類型動態調整
- [ ] 圖片上傳/拍照選項正常工作

### 集成測試
- [ ] 回答內容正確存儲至資料庫
- [ ] 隔天自動更新題目
- [ ] 題庫同步不影響已記錄數據
- [ ] 離線時仍可查看已快取題目

## 非功能需求

- **效能**：題目加載 < 1 秒
- **可用性**：最多 3 個操作步驟完成回答
- **一致性**：同一天內所有使用者看到相同的 2 個題目（提升社群感）
- **離線支援**：已快取題目在無網路時可用

## 數據儲存

### SwiftData
- 本地存儲回答內容
- 題目快取（最新 100 個題目）
- 同步狀態標記

### Firestore
- 全球題庫（中央真實來源）
- 題庫版本控制
- 使用者回答統計（匿名聚合）

## 相依項

- 題庫完整度（18 × 多題目 ≥ 540 道）
- Firestore 連接穩定
- 本地存儲空間 ≥ 50MB
