# AI 聊天日記功能規格

## 基本資訊
- **功能名稱**：AI 聊天日記（AI Chat Diary）
- **版本**：v1.1
- **狀態**：實作中
- **優先度**：高
- **建立日期**：2026-05-31

## 目的
降低使用者的日記書寫門檻，透過與 AI 進行自然對話，系統自動將對話內容總結為正式心理紀錄，讓使用者無需思考結構化問題，只需簡單傾訴即可產出完整的情緒記錄。

## 範圍

### In-Scope（包含）
- 即時 AI 對話介面
- 自動對話摘要生成
- 對話內容本地存儲
- 對話歷史管理
- 生成的日記內容編輯

### Out-of-Scope（不包含）
- 多語言支援（階段二）
- 語音輸入（階段二）
- 對話分享功能（階段二）

## 使用者故事

### 故事 1：快速傾訴
**As a** 疲憊的上班族  
**I want to** 下班後能快速傾訴今天的情緒，無需結構化思考  
**So that** 我能快速宣洩情感並記錄心情

### 故事 2：主動對話式日記
**As a** 常常不知道寫什麼的使用者  
**I want to** AI 主動引導我回答問題，幫助我整理思緒  
**So that** 我能更深入地反思自己的情緒

### 故事 3：編輯生成的日記
**As a** 重視隱私的使用者  
**I want to** 在 AI 生成日記後可以編輯或刪除敏感內容  
**So that** 我對最終記錄的內容有完全控制

## 功能流程

```
1. 使用者點擊「AI 聊天日記」
2. 進入聊天介面
3. AI 主動開啟對話，例如：「今天怎麼樣？想和我分享一下嗎？」
4. 使用者輸入回應
5. AI 根據使用者回應進行深層提問，引導更深入的反思
6. 對話持續直至使用者結束或達到一定長度
7. 使用者點擊「生成日記」
8. 系統呼叫 Gemini API 進行摘要
9. 生成的日記內容顯示供編輯
10. 使用者確認後，日記儲存至本地與雲端
```

## 驗收標準

- [ ] 使用者能點擊進入 AI 聊天介面
- [ ] AI 能從預設開場白開始對話
- [ ] 使用者能輸入文字與 AI 互動
- [ ] 對話內容即時顯示在聊天泡泡中
- [ ] 生成按鈕點擊後，系統呼叫 API 並在 3 秒內返回摘要
- [ ] 生成的日記內容可編輯（文字可修改、刪除或補充）
- [ ] 確認後日記內容正確存儲至 SwiftData 與 Firestore
- [ ] 對話歷史被正確清除或封存
- [ ] API 呼叫失敗時顯示友善的錯誤提示

## 技術實現

### 使用的 API / 服務
- **Gemini API**：自然語言處理與摘要生成
- **SwiftData**：本地存儲
- **Firestore**：雲端備份

### 資料結構
```swift
struct ChatMessage {
    let id: UUID
    let content: String
    let isFromUser: Bool
    let timestamp: Date
}

struct AIGeneratedDiary {
    let id: UUID
    let originalChatMessages: [ChatMessage]
    let generatedContent: String
    let createdAt: Date
}
```

### API 呼叫流程
1. 收集所有聊天訊息（使用者 + AI）
2. 組織成提示詞：「請根據以下對話內容生成一篇簡潔的情緒日記摘要...」
3. 呼叫 Gemini API
4. 接收摘要並顯示

## 測試要點

### 單元測試
- [ ] 聊天訊息正確新增至陣列
- [ ] 摘要生成邏輯正確
- [ ] 日記內容正確保存至資料庫

### UI 測試
- [ ] 聊天泡泡正確排列（使用者在右，AI 在左）
- [ ] 輸入框可聚焦並接收文字
- [ ] 生成按鈕在有對話內容時可用
- [ ] 編輯介面允許修改內容

### 集成測試
- [ ] API 呼叫成功時返回摘要
- [ ] API 呼叫失敗時顯示重試選項
- [ ] 日記正確保存至本地與雲端

## 非功能需求

- **效能**：API 回應時間 < 5 秒
- **可用性**：最多 3 個操作步驟完成一篇日記
- **隱私**：對話內容不涉及第三方伺服器存儲（摘要後立即刪除）
- **離線**：已對話的內容在無網路時可本地保存，重新連接後同步

## 相依項

- Apple HealthKit 數據整合（可選，用於後續週報）
- Firestore 連接正常
- Gemini API 配額充足

## 風險與緩解

| 風險 | 影響 | 緩解方案 |
|------|------|--------|
| API 配額超限 | 無法生成摘要 | 實施速率限制與本地快取 |
| 摘要質量不佳 | 使用者不滿意 | 提供編輯功能與反饋機制 |
| 網路延遲 | 使用者體驗差 | 顯示加載狀態與逾時處理 |
